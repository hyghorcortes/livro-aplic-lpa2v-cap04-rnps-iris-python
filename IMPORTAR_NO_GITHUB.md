# Como publicar este projeto no GitHub

## Nome sugerido do repositório

```text
livro-aplic-lpa2v-cap04-rnps-iris-python
```

## Descrição curta sugerida

```text
Exemplo em Python do Capítulo 4 — RNPs do livro Aplicações de LPA2v, aplicado à classificação da base Iris.
```

## Topics sugeridos

```text
lpa2v, paraconsistent-logic, paraconsistent-neural-networks, rnp, iris-dataset, python, machine-learning, book-example
```

## Opção A — Criar no GitHub e subir pela interface web

1. Crie um novo repositório vazio no GitHub.
2. Use o nome sugerido acima.
3. Não adicione README, `.gitignore` ou licença na tela inicial, porque estes arquivos já estão neste pacote.
4. Após criar o repositório vazio, envie o conteúdo desta pasta.
5. Faça o commit inicial com uma mensagem objetiva, por exemplo:

```text
Initial commit: Capítulo 4 RNPs Iris em Python
```

## Opção B — Criar pelo Git local

Dentro desta pasta, execute:

```bash
git init
git branch -M main
git add .
git commit -m "Initial commit: Capítulo 4 RNPs Iris em Python"
git remote add origin https://github.com/hyghorcortes/livro-aplic-lpa2v-cap04-rnps-iris-python.git
git push -u origin main
```

## Sobre o que já está pronto neste pacote

Este pacote já inclui:

- documentação principal do projeto;
- requisitos Python;
- script para executar todos os exemplos;
- licença;
- metadados de citação;
- arquivos de apoio à colaboração;
- estrutura de repositório organizada.

## Ajustes finais recomendados no GitHub

Depois de publicar, ajuste no campo **About**:

- descrição curta;
- URL do projeto ou do livro, se houver;
- topics sugeridos;
- imagem social do repositório, se desejar.
