# Política de segurança

## Como reportar vulnerabilidades

Se você identificar uma vulnerabilidade ou problema de segurança relevante, envie um relato privado aos mantenedores antes de divulgar publicamente.

Inclua, quando possível:

- descrição do problema;
- impacto potencial;
- passos para reprodução;
- arquivos afetados.

## Escopo

Este repositório é acadêmico e didático, mas ainda assim deve evitar práticas inseguras em código, dependências, arquivos gerados ou automações.
